## 关键发现根因分析

F1 和 F2 共享同一个根因：**在 `axil_slave_if.sv` 中，写数据的捕获（capture）与转发（forwarding）被安排在同一个时钟周期的不同 always 块中，但 SystemVerilog 的非阻塞赋值语义使得转发逻辑在捕获之前读取了旧值**。

具体来说，以下两个 always 块在 `posedge clk` 时同时触发：

```
块1: if (w_hsk) wdata_q <= s_axil_wdata;   // 捕获总线数据
块2: if (w_exec) wdata <= wdata_q;           // 转发到 regs_top
```

在 NBA 语义下，两个块的 RHS（`s_axil_wdata` 和 `wdata_q`）在时钟上升沿时（即在任何 LHS 更新之前）被同时采样。因此块2读到的 `wdata_q` 是块1更新前的旧值。

**验证方法**：编写一个简单的测试向量，连续执行两次写事务（先写 IMG_ROWS=0x100，再写 IMG_COLS=0x200），观察 regs_top 内部寄存器值即可发现 IMG_COLS 得到的是 0x100（前一次的值）而非 0x200。

**修复简要方案**：在 forwarding 逻辑中使用组合旁路：

```verilog
// 在 axil_slave_if 中增加组合旁路
wire [31:0] wdata_comb = w_hsk ? s_axil_wdata : wdata_q;
wire [3:0]  wstrb_comb = w_hsk ? s_axil_wstrb : wstrb_q;
wire [5:0]  awaddr_comb = aw_hsk ? s_axil_awaddr[7:2] : awaddr_q[7:2];

// 转发时使用组合值
always @(posedge clk) begin
    ...
    if (w_exec) begin
        if (awaddr_comb < 6'd16)
            wr_strobe <= 16'd1 << awaddr_comb;
        wdata <= wdata_comb;
        wstrb <= wstrb_comb;
    end
end

// bresp 也使用组合值
always @(posedge clk) begin
    if ((wstate_n == W_RESP) && (wstate != W_RESP)) begin
        bresp_q <= (awaddr_comb < 6'd16) ? 2'b00 : 2'b10;
    end
end
```
