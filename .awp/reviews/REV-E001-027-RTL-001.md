## 详细审查要点逐项分析

### 1. dbg_trigger_hub.sv 设计正确性

**FSM 边沿检测：** 正确。
- `fsm_prev_idle` 复位值 `1'b1`，与 FSM 复位后 IDLE 状态一致，不会产生复位后误触发。
- `fsm_start_edge = fsm_prev_idle && !fsm_idle`：IDLE->CAPTURE 时产生正脉冲，逻辑正确。
- `shift_start_edge = fsm_prev_capture && !fsm_capture`：CAPTURE->SHIFT 时产生正脉冲，逻辑正确。
- DONE->IDLE 边沿不作为 anchor event，符合需求。

**AXIS 第一个 beat 检测：** 正确。
- `axis_first_beat = capture_en && axis_tvalid && axis_tready && !axis_first_beat_seen`
- `axis_first_beat_seen` 在 capture_en=1 且握手发生时在时钟沿置 1，组合逻辑在同一周期检测到非组合反馈的旧值，因此第一个 beat 能被正确检测到。
- 在 `capture_en=0` 时清零（不是仅靠复位），确保多帧场景下每帧都能重新检测。正确。

**触发脉冲展宽（1 -> 4 cycles）：** 正确。
- `pulse_stretch` 是 3-bit shift register，当 `anchor_raw` 为 1 时加载 `3'b111`，随后每周期右移补 0。
- `trig_stretched = |pulse_stretch` 保持 4 周期高（cycle 0: 111, cycle 1: 011, cycle 2: 001, cycle 3: 000）。
- ILA 的 `trig_in` 最小脉冲宽度在 Vivado ILA 文档中为 1 周期，4 周期展宽留有充分裕量。正确。

**trig_sel 多路选择：** 正确，但复用 3 返回 1'b0（安全 fallback）。未实现 `axis_error`（预留）。

**自由运行计数器：** 正确，32-bit，复位后从 0 开始计数，永不停止。

### 2. axil_2d_shift.sv ILA 例化

**端口连接检查：**

| ILA 端口 | 连接信号 | 评估 |
|----------|----------|------|
| clk | clk（100 MHz 系统时钟） | 正确，与逻辑同频采样 |
| trig_in | dbg_trig_pulse | 正确，来自 dbg_trigger_hub |
| trig_in_ack | trig_in_ack_ctrl/trig_in_ack_data | 声明但未使用，可接受但浪费了诊断信息 |
| probe0 | dbg_cycle_cnt[31:0] | 32-bit，正确 |
| probe1 | {20'd0, fsm_state, ctrl_start, ...} | **33-bit，见 F2** |
| probe2 | {22'd0, img_rows[9:0]} | 32-bit，正确 |
| probe3 | {22'd0, img_cols[9:0]} | 32-bit，正确 |
| probe4 | {27'd0, status_idle, ..., capture_en} | 32-bit，正确 |
| probe5 | {30'd0, shift_en, shift_done} | 32-bit，正确 |
| probe6 | 32'd0 | 占位，正确 |
| probe7 | {30'd0, dbg_anchor_status, dbg_trig_pulse} | 32-bit，正确 |

**data_cross 各 probe 信号分配：** 所有 probe 宽度检查通过，均为匹配的单信号或 8/12-bit 总线，宽度匹配风险低。

### 3. Tcl 脚本

**arm -> wait -> PS start 流程：** 架构逻辑正确 —— 先 arm 两个 ILA，确认状态为 Waiting for Trigger，再启动 PS。这保证 PS 启动产生的第一个事务不会被 ILA 遗漏。

**TRIG_IN_ONLY 模式配置：** 正确。TRIGGER_MODE=TRIG_IN_ONLY 意味着 ILA 仅在 trig_in 脉冲到达时触发，数据捕获模式为 ALWAYS（持续采集）。DATA_DEPTH 2048/4096 分配合理（ctrl 面用较浅深度，data 面用较深）。

**数据导出：** csv + .ila 双格式，满足分析需求。

### 4. 整体架构

- 默认 anchor event（fsm_start_edge）选择合理 —— FSM 从 IDLE 进入 CAPTURE 是最明确的事件边界。
- 多 ILA 同步触发机制简化了同步问题：所有 ILA 用同一个 dbg_trig_pulse 同时触发，trig_in_ack 可用于确认每个 ILA 都已收到触发。
- dbg_cycle_cnt 作为公共时间基准，多个 ILA 捕获的数据可通过 cycle count 交叉比对。
