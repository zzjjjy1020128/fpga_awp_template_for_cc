## Detailed Review

### State Machine Completeness
- 状态编码使用 `typedef enum logic [1:0]`，覆盖全部 4 种 2-bit 组合（00/01/10/11），无非法编码。
- `default` 分支将未知状态导向 `IDLE`，可抵御 SEU 导致的非法状态。

### State Transitions
- IDLE: `ctrl_start` → CAPTURE, `sw_reset` 保持 IDLE — 与架构一致。
- CAPTURE: `capture_done` → SHIFT; `sw_reset` → IDLE — 与架构一致。
- SHIFT: `shift_done` → DONE; `sw_reset` → IDLE — 与架构一致。
- DONE: 无条件（auto）→ IDLE; `sw_reset` → IDLE — 与架构"保持 1 周期后自动返回"一致。

### sw_reset Priority
- 每个状态中 `ctrl_sw_reset` 均在其他转换条件之前判断，优先级正确。
- IDLE 中 sw_reset 保持 IDLE（而非无效跳转），行为合理。

### Output Mutual Exclusivity
- 输出译码基于 `state` 比较，由于 `state` 为枚举单值，四个状态标志（idle/busy_capture/busy_shift/done）天然互斥。
- `capture_en`（CAPTURE 状态）和 `shift_en`（SHIFT 状态）同样互斥。

### Code Quality
- 三段式结构清晰：`always_comb` 次态、`always_ff` 状态寄存、`always_comb` 输出译码。
- 次态逻辑使用 `next = state` 默认赋值 + `case` 覆盖的 pattern，无 latch 推断风险。
- 输出译码所有信号无条件赋值，无 latch 风险。
- 所有输入端口被使用，所有输出端口被驱动，无未连接/未驱动信号。
