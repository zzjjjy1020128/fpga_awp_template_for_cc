## Detailed Analysis

### Finding 1（核心问题）：Wrap 模式模运算正确性

**问题根源**：architecture.md 3.6 节公式中使用了 `%` 运算符，但 RTL 出于综合考虑将其替换为 compare-and-adjust（CAA）优化。CAA 仅在 `dividend < 2*divisor` 时正确。由于 `step` 为 5-bit（0~31），`img_rows` / `img_cols` 为 10-bit（实际 1~64），`row_cnt + step` 可达 `img_rows + 30`，当 `img_rows` 较小时 (`< 32`)，该值可能超过 `2*img_rows`，CAA 失效。

**各模式影响分析**：

| 模式 | 当前 RTL | 失效条件 | 示例（失效时） |
|------|---------|---------|--------------|
| UP wrap | `(r+s >= R) ? (r+s-R) : (r+s)` | `s >= R` | R=10, s=15, r=8: 得 13, 正确 3 |
| DOWN wrap | `(r < s) ? (r-s+R) : (r-s)` | `s > r+R`（即 `s > R` 起点） | R=10, s=15, r=0: 得 1009, 正确 5 |
| LEFT wrap | 同 UP（col 维度） | `s >= C` | C=10, s=15, c=8: 得 13, 正确 3 |
| RIGHT wrap | 同 DOWN（col 维度） | `s > C` | C=10, s=15, c=0: 得 1009, 正确 5 |

（R=img_rows, C=img_cols, r=row_cnt, c=col_cnt, s=step）

**非 wrap 模式不受影响**：当 `s >= R` 时所有像素标记为 zero_fill，行为正确。

**建议修复方案**：在计算式中直接使用 `%` 运算符。

UP wrap:
```systemverilog
calc_row = (row_cnt + step) % img_rows;
```

DOWN wrap（需处理负数取模）:
```systemverilog
calc_row = (row_cnt + img_rows - (step % img_rows)) % img_rows;
// 等价的 CAA 版本（保证 step < img_rows 时正确）：
// calc_row = (row_cnt + img_rows - step) % img_rows;
```

或使用两次 CAA（处理 step < 2*img_rows 的常见情况）：
```systemverilog
// UP wrap
tmp = row_cnt + step;
calc_row = (tmp >= img_rows) ? (tmp - img_rows) : tmp;
calc_row = (calc_row >= img_rows) ? (calc_row - img_rows) : calc_row;

// DOWN wrap
calc_row = (row_cnt >= step) ? (row_cnt - step) : (row_cnt + img_rows - step);
calc_row = (calc_row >= img_rows) ? (calc_row - img_rows) : calc_row;
```

### Finding 2：复位风格

architecture.md 规格为同步复位，实现为异步复位。虽 FPGA 工具对两种复位风格均支持，但：
- 同步复位资源利用率略低（不需专用复位布线）
- 架构文档为契约，实现应与之一致

### 计数器逻辑正确性确认

- `shift_en=0` 时计数器保持：通过 `else if (shift_en)` 实现，正确。
- Rollover：col 到边界后归零且 row 加 1；row 到边界后归零。符合光栅扫描。
- `img_cols=1` 时每拍 col 归零、row 递增 —— 行为正确。
- `img_rows=1` 时 row 始终为 0 —— 行为正确。
- 不存在组合逻辑环路：`read_addr` 仅依赖寄存器输出的 `row_cnt/col_cnt` 和输入 `img_cols`。

### read_addr 乘法宽度

`calc_row * img_cols + calc_col` 中：
- `calc_row` : 10-bit（最大 1023）
- `img_cols` : 10-bit（实际最大 64）
- 乘积：最大 63*64=4032, 13-bit
- 加 `calc_col`（最大 63）后：最大 4095, 12-bit
- `read_addr` 为 12-bit，常规范围充足
- Finding 1 中的模运算错误会导致 calc_row 超出 63，进而 read_addr 溢出截断，产生随机地址
