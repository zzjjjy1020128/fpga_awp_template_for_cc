## 详细审查分析

### 1. AXI-Stream 输出握手协议 (tvalid/tready)

- `m_axis_tvalid = shift_en && !all_done`：当 shift_en=1 且未完成时持续有效，不依赖 tready（符合 AXI-Stream 规范 "TVALID must not depend on TREADY"）。
- 计数器仅当 `m_axis_tready && !all_done` 时推进，正确实现背压暂停。
- `m_axis_tvalid` 在最后一拍握手后的下一个周期自动降为 0。

**结论：正确。**

### 2. Backpressure 处理

- 当 `m_axis_tready=0` 时，`if (m_axis_tready && !all_done)` 条件不满足，计数器保持当前值。
- 外部 `read_data` 由 `shift_addr_gen` 驱动地址，而 `shift_addr_gen` 的输入 `row_cnt`/`col_cnt` 在此阶段不变，因此 `m_axis_tdata` 在背压期间保持稳定。
- 对最后一拍的背压也能正确处理：`all_done` 只有在成功握手后才置位。

**结论：正确。**

### 3. zero_fill 信号处理

- `assign m_axis_tdata = zero_fill ? {DATA_WIDTH{1'b0}} : read_data;`
- `zero_fill=1` 时强制输出全 0，不影响 `tvalid`/`tlast`/`tuser` 生成。
- `zero_fill` 由 `shift_addr_gen` 同步驱动，与 `row_cnt`/`col_cnt` 变化同步，无时序违例风险。

**结论：正确。**

### 4. shift_done 脉冲生成时机

- `shift_done` 在最后一拍成功握手（`col_cnt == img_cols-1 && row_cnt == img_rows-1 && tready=1` 且 `!all_done`）时同步置位。
- 下一周期通过默认赋 `shift_done <= 1'b0` 自动清零（procedural block 中最后赋值语义）。
- 生成 1 周期脉冲，符合设计要求。
- 注意：`shift_done` 与最后一拍数据在同一时钟周期内同时有效。若 `ctrl_fsm` 在接收到 `shift_done` 后立即撤销 `shift_en`，需确保 `ctrl_fsm` 在同一周期内不会因 `shift_en` 变化而误判。

**结论：正确。**

### 5. 帧边界 (tuser/tlast) 处理

- **tuser**：`row_cnt == 0 && col_cnt == 0`，仅在帧的第一个像素（左上角）拉高，符合 architecture.md 定义。
- **tlast**：`col_cnt == img_cols - 1`，每行最后一个元素拉高，每帧共 `img_rows` 次 tlast 脉冲，符合 architecture.md 定义。

**结论：正确。**
